The Scar Programming Language

Program Structure

Programs in the Scar language consist of modules.
Each module has a name and consists of global constants and functions.
The module name must begin with a lowercase Latin letter, and the remaining characters (if any) must be either lowercase Latin letters, digits (0-9), or underscores (_).
If the module name is main, then that module must contain a function named main, which is the one executed in the resulting program.
To create a module using the scarlet compiler, the following must be executed:

scarlet -n module_name -s source_code_file_1 source_code_file_2 source_code_file_n -o output_module_file

If the module being created uses other modules, the already created module files are passed using the -m flag, for example:

scarlet -n module_name -m module_file_1 module_file_2 module_file_n -s source_code_file -o output_module_file

Source code files have the extension scar, and module files have the extension mscar.
If the module has the name main, then instead of a module file, an intermediate representation file is saved. Currently, this is source code in the language C intended only for the clang compiler.

Structure of Source Code Files

A source code file may contain the following elements:
  • Single-line comment. Starts with the # character and ends at the end of the line.
  • Multi-line comment. Starts with the #* characters and ends with the *# characters.
  • Documentation comment. Starts with the #@ characters and ends with the @# characters. It is placed before a global constant or function (but before its attributes).
  • Function or global constant attributes. Placed before the global constant or function.
  • Global constant. (Details in the Global Constants section)
  • Function. (Details in the Functions section)
Tab characters are not allowed for code formatting.
Source code must be in ctf8 encoding. (ASCII compatible)

Global Constants

A global constant is a constant with a name, the value of which is accessible by that name in expressions of other constants and functions. (For details on expressions, see the Expressions section)
Declaration of a constant in source code: 
`attributes`
let constant_name = value
Specifying attributes is not mandatory.
Attributes:
  • export - the constant is accessible from any module; without this attribute it is only accessible within the module where it is declared.
The constant name must begin with a lowercase Latin letter or an underscore (_), and the remaining characters (if any) must be either lowercase Latin letters, digits (0-9), or underscores.
A constant name cannot consist of a single underscore.
Within one module, two constants cannot have the same name.

The value is specified as an expression whose result is the value of the constant.
If the expression cannot be evaluated at compile time, it is evaluated before the execution of the main function from the main module.
The expression cannot use threads.
The value of a global constant is evaluated only if the main function from the main module, directly or indirectly, uses that constant.
If the value of a global constant has type error, the program immediately stops with a message about this error.

Functions

A function is an expression in which constants can be referenced by name, and the values of these constants will be specified when the function is called. (Example: fn y(x) x + 1) (For details on expressions, see the Expressions section)
The constants whose values are specified when calling the function are called function arguments, and their names inside the function are called function parameters.
The expression is called the function body.
The function body is evaluated when the function is called, replacing the function parameters with the evaluated argument values.
Arguments are evaluated in the same order they are passed to the function.
Declaring a function in source code: 
`attributes`
fn function_name(parameter_1, parameter_2, parameter_n) expression
Specifying attributes is not mandatory.
If it is necessary to list several attributes, they are separated by one or more spaces.
Attributes:
  • cold - the function is optimized as if it will never be executed, but if it does execute, it will work correctly.
  • export - the function is accessible from any module; without this attribute it is only accessible within the module where it is declared.
  • external - the implementation of this function is made in another programming language. In this case, the function body is not specified.
  • hot - the program is optimized as if this function will be executed frequently. Specifying this attribute on a function that is rarely called may lead to degraded program performance.
  • noinline - sometimes a function call can be replaced with the function body with the arguments substituted; this attribute disables such behavior for this function.
The hot and cold attributes cannot be specified simultaneously.

The function name must begin with a lowercase Latin letter or an underscore (_), and the remaining characters (if any) must be either lowercase Latin letters, digits (0-9), or underscores.
Also, one of the following suffixes is allowed in the function name - !, !!!, ?.
Meaning of suffixes in function names:
  • ? - the function name is a question, and the result of the function evaluation will give an answer to it.
  • ! - the function name is an assertion, and if the assertion is not true, the function will change the state of the application so that it becomes true, or return an error.
  • !!! - the function is unsafe and should be used with caution.
The function name cannot be one of the names from the following list: case, default, else, fn, guard, if, let, new, not, scarlet__erase_info, scarlet__erase_value, switch, test, unreachable, _
Within one module, two functions cannot have the same name.

A function parameter name must begin with a lowercase Latin letter or an underscore (_), and the remaining characters (if any) must be either lowercase Latin letters, digits (0-9), or underscores.
A function parameter name cannot be one of the names from the following list: case, default, else, false, fn, if, let, new, not, null, switch, true
If _ is specified instead of a parameter name, it means that the function does not need the value of that argument.
Within one function, two parameters cannot have the same name.
The number of parameters in a function must not exceed 16.

Expressions

An expression is a computation or an action that returns a result.
If an expression is expected somewhere in the source code, any number of newline characters or ; characters may precede it (except when these characters replace a comma). (Example: 2 + ;; 2 , the same as 2 + 2 )
Expressions have a precedence (a number from 0 to 10).
If an expression consists of several expressions, then that expression is the one with the higher precedence.
If there are two expressions with the same precedence, the expression closer to the beginning of the source expression is considered to have lower precedence.
For simplicity, further in the text, the process of determining which expression a given expression is will be called "determining the root expression". Example:
1 + 2 + 4
This expression is a binary function + (on binary functions, further), which takes two arguments: the first 1 + 2, the second 4.

List of possible expressions:
  • Binary function
    Binary functions are functions that take two arguments, with the function name placed between them.
    It is not necessary to pass all arguments to the function at once; you can pass part of the arguments, and in place of the rest pass _, in which case the function call returns a function that takes as many arguments as the number of _ passed, and then executes the original function replacing _ with the passed arguments in the same order. Such application of a function is called partial.
    In partial application, the passed arguments cannot be of the following types: box, breaker, thread, error. (Details on types in the Types section)

    Examples:
    _ + 1 , the same as fn (a) a + 1
    _ + _ , the same as fn (a, b) a + b

    List of binary functions and their precedence:
    Functions Precedence
    [] : [,] 0
    * / % 3
    + ++ - 4
    << >> >>> 5
    < > <= >= 6
    == /= === =/= 7
    & 8
    ^ 9
    | 10
    The actions performed by these functions are described in the documentation for the core module.

  • Binary operator
    Binary operators are the same as binary functions, but without the possibility of partial application and with lazy evaluation of arguments.
    List of binary operators and their precedence:
    Operator Precedence Action performed
    <? 1 Evaluates the left expression; if it is null, evaluates the right expression and its result is the result of the entire expression, otherwise does not evaluate the right expression and the result of the entire expression is the result of the left expression. (Details on null in the Types section)
    && 8 Evaluates the left expression; if it equals true, evaluates the right expression and its result is the result of the entire expression, otherwise does not evaluate the right expression and the result of the entire expression is false. Both expressions must have type bool. (Details on false, true and the type bool in the Types section)
    || 10 Evaluates the left expression; if it equals false, evaluates the right expression and its result is the result of the entire expression, otherwise does not evaluate the right expression and the result of the entire expression is true. Both expressions must have type bool. (Details on false, true and the type bool in the Types section)

  • Function call
    A function can be called in source code in two ways:
    1. function_name(argument_1, argument_2, argument_n)
    2. argument_1.function_name(argument_2, argument_n)
    The function name can be specified in two ways:
    1. short - simply the function name. Allowed only if there is a function with the specified name in exactly one module.
    2. full - the function name is specified in the format @module_name.function_name.
    As with binary functions, partial application can be used.
    If there are multiple arguments, the comma between them can be replaced by a newline character or a ; character.
    Example:
    foo(1, 2, 3, 4)
    the same as 
    foo(
     1
     2; 3, 4
)
    Arguments passed after specifying the function name (in parentheses) do not participate in the same root expression determination as the function call itself.
    Precedence - 0.

  • check operator
    The operator works like a function, but without the possibility of partial application.
    The operator takes one argument, and if it is of type error, the application immediately stops with information about where in the source code the operator was used and what the error in the argument was; otherwise, the result of the operator is its argument. (Details on the type error in the Types section)

    Precedence - 0.

  • guard operator
    The operator works like a function, but without the possibility of partial application.
    The operator takes one argument, and if it equals true, the result of the operator is true, otherwise the application immediately stops with information about where in the source code the operator was used. (Details on true in the Types section)

    Precedence - 0.

  • test operator
    The operator works like a function, but without the possibility of partial application.
    The operator takes one argument.
    If there has been at least one use of the test operator, then after the application finishes, information is displayed about how many times the operator was called, how many times the argument was equal to true, as well as information about where in the source code the operator was used with an argument other than true. The result of the operator is null. Details on true and null in the Types section.

    Precedence - 0.

  • unreachable operator
    The operator takes no arguments.
    When the operator is called, the application immediately stops with information about where in the source code the operator was used.

    Precedence - 0.

  • Constants true and false (Details in the Types section)

    Precedence - 0.

    (Details in the Types section)

    Precedence - 0.

    (Details in the Types section)

    Precedence - 0.

    (Details in the Types section)

    Precedence - 0.

    (Details in the Types section)

    Precedence - 0.

  • Token
    (Details in the Types section)

    Precedence - 0.

  • Global constant
    Obtaining the value of a global constant in source code:
    @module_name.constant_name

    Precedence - 0.

  • Grouped expression
    Several expressions can be grouped into a single expression; in this case, the expressions are executed in the order they are specified, and the result of the grouped expressions is the result of the last expression.
    To group expressions, all expressions must be enclosed in curly braces ({}), separating expressions with a newline character or a ; character.
    In grouped expressions, you can bind a name to a constant by writing as one of the expressions let constant_name = value, after which, until the end of the grouped expression, the result of the expression will be accessible by that name.
    If another constant was already bound to that name, or that name is a function parameter, then until the end of the grouped expression, the new constant will be accessible by that name.
    The rules for constant names are the same as for function parameters.
    Names cannot be bound to constants in the last expression.
    You can also bind constants to names using unpacking; details on unpacking in the Types section.

    Grouped expressions participate in determining the root expression as a single whole.
    Precedence - 0.

  • Local constant
    A local constant is a constant bound to some name, but not belonging to any module.
    Function arguments within functions themselves are local constants.
    Constants that were bound to a name in grouped expressions are local constants.
    To access the value of a constant, you simply specify its name.
    Precedence - 0.

  • Nested expression
    An expression can be wrapped in parentheses (), in which case the expression participates in determining the root expression as a single whole.
    Precedence - 0.

  • new operator
    If some constant stores other constants within it, then using the new operator you can create a new constant, the same as that constant, but with new values for the inner constants.
    The new operator has two forms:
    1. new (constant)[key_1][key_2][key_3] modifier value
    2. new (constant) {[key_1][key_2][key_3] modifier_1 value_1; [key_4][key_5][key_6] modifier_2 value_2}
    The constant on the basis of which the result will be created is written in parentheses after new.
    Keys are values with which it is determined which part of the constant will be changed; exactly how this happens depends on the type to which the key is applied.
    (Details in the Types section)
    If a token is used as a key for the new operator directly in the form :token, the square brackets ([]) may be omitted.
    (Details on tokens in the Types section)
    key_1 indicates which part of the constant will be changed, and key_2 which part of the constant pointed to by key_1 will be changed, etc.

    Modifier types:
    Modifier Purpose
    <= replaces the old value with the new one
    <<= same as <=, but the operator returns not only the new value of the constant, but also the old value of the changed part (values wrapped in type box, details in the Types section)
    => same as <=, but as the new value a function is specified which takes the old value of the part being changed as an argument and returns its new value
    =>> same as =>, but the operator returns not only the new value of the constant, but also the old value of the changed part (values wrapped in type box, details in the Types section)

    The expression whose result is the new value of the inner constant is taken as long as it is valid. Example:
    new ([array x])[0] <= 12 + 1
    This expression will return [array 13]

    Evaluation order of expressions:
    1. keys
    2. new values
    3. constant

    In the second form of the operator, the semicolon can be replaced by a newline character.
    The second form of the operator is a convenient form for writing several consecutive uses of the first form of the operator. Example: 
    new (a) {
     [1] <= 2
     [f][x][13] <= f
}
    the same as 
    new (new (a)[1] <= 12)[f][x][13] <= f
    In the second form of the operator, the modifiers <<= and =>> can only be used in the last modification.

    The new operator participates in determining the root expression as a single whole.
    Precedence - 0.

  • if operator
    The if operator is used to evaluate one of two expressions.
    Operator in source code:
    if predicate expression_1 else expression_2

     To separate the predicate and expression_1, a newline character or a ; character must be used.
    If expression_1 is a grouped expression, separating predicate and expression_1 is not necessary.
    Any number of newline or ; characters may appear before else.
    The expression after else is taken as long as it is valid.

    The operator evaluates the predicate; if its result equals true, then expression_1 is evaluated and its result is the result of the entire expression, otherwise expression_2 is evaluated and its result is the result of the entire expression. The predicate must have type bool. (Details on true and the type bool in the Types section)

    The if operator participates in determining the root expression as a single whole.
    Precedence - 0.

  • switch operator
    The switch operator is used to evaluate one of several expressions.
    Operator in source code:
    switch expression_0 case value_1, value_2: expression_1 case value_m: expression_n: default: expression_d

    The operator evaluates expression_0; if its result equals one of the values following case up to the colon, then the expression after the colon is evaluated and its result is the result of the entire expression; otherwise the expression after default: is evaluated and its result is the result of the entire expression.
    After case, several expressions may appear in a row, separated by a comma (which can be replaced by a newline character or a ; character).
    The expression after default: is taken as long as it is valid.
    For all expressions after case, it must be possible to compute a hash sum using the hash function from the core module, and equality comparison is performed using the binary function ===.
    Within one switch operator, for any two expressions after any case, the binary function === must return false. (Details on false in the Types section)
    If the expression after case cannot be evaluated at compile time, it is evaluated before the execution of the main function from the main module.

    The switch operator participates in determining the root expression as a single whole.
    Precedence - 0.

  • Lambda function
    A lambda function is a function without a name.
    A lambda function differs from a regular function only in the absence of a name and the absence of belonging to any module.
    Lambda functions in source code:
    fn (parameter_1, parameter_2, parameter_n) expression

    Calling a lambda function in source code:
    (expression)(argument_1, argument_2, argument_n)

    For lambda functions, as for regular functions, partial application can be used.
    In the body of a lambda function, you can refer to local constants that were declared outside the lambda function; such constants cannot be of the following types: box, breaker, thread, error. (Details on types in the Types section)
    If the body of a lambda function references local constants that were declared outside the lambda function and there is a parameter of that lambda function with the same name, then in that case the name will be interpreted as the function parameter.
    The expression is taken as long as it is valid.

    A lambda function participates in determining the root expression as a single whole.
    Precedence - 0.

  • Method call
    (Details in the Types section)

    Precedence - 0.

  • not function The function in source code:
    not expression

    The function evaluates the expression; if its result equals true, the function result is false, and if the expression result equals false, the function result is true.
    The function accepts only arguments of type bool. (Details on false, true and the type bool in the Types section)

    Precedence - 1.

Types

A type is a characteristic of a constant, based on which the program determines what that constant is and how to work with it.
List of types:
  • string
    string - text.
    The text may contain no characters, such a string is called empty.
    Strings in the scar language use their own character encoding called corsar.
    A string begins and ends with the " character.
    Strings support escape sequences, which begin with the \ character.
    Escape sequences:
    escape sequence meaning
    \n newline character
    \t tab character
    \' the ' character
    \" the " character
    \\ the \ character
    \xxxxxx character with code xxxxxx, where xxxxxx is the hexadecimal character code in the corsar encoding

    Before the opening quote, the letter r can be specified, in which case escape sequences will be ignored.
    Before the opening quote, the letter u can be specified, in which case the string will have type byte_array (on that later), and not string, and will store bytes of the string analogous to this one but in the utf8 encoding, and the escape sequence \xxxxxx will mean the code in Unicode.
    If the letter u is specified, the letter z can be specified after it, in which case a byte 0 (zero) will be added to the end of the string.
    Before u or uz, r can be specified, in which case escape sequences will be ignored.

    String examples: 
    ""
"Hello !!!"
"Char \000100"
"New line \n"
r"not new line \n"
"\""
u""
uz""
u"Hello"
u"\000100"
ru"\000100"
ruz"\000100"
    To change a character, the new operator is used; the index of the character (starting from 0) is passed as the key, and the new character is passed as the value.

  • char (character)
    char - a single text character.
    A character begins and ends with the ' character.
    Characters support escape sequences in the same way as strings.
    Before the opening quote, the letter u can be specified, in which case the character will have type int (on that later), and not char, and stores the Unicode code point of a character analogous to this one, and the escape sequence \xxxxxx will mean the code in Unicode.
    Character examples: 
    'a'
'z'
' '
'\000100'
'\n'
'\''
u'a'
u'H'
u'\000100'
  • int
    int - an integer from the set [-9'223'372'036'854'775'808 - 9'223'372'036'854'775'807] (-2^63 - 2^63-1).
    When writing a number, the ' character can be used as a digit separator.
    A positive number may be preceded by the + character.
    If during calculations a number goes outside the set of allowed numbers, arithmetic overflow occurs.
    A number can be written in hexadecimal notation, for which 0x must be written before the number (after the - or + characters, if present).
    Hexadecimal notation can have values from -0xffff'ffff'ffff'ffff to 0xffff'ffff'ffff'ffff (-(2^64-1) - 2^64-1), but if the number does not belong to the set [-9'223'372'036'854'775'808 - 9'223'372'036'854'775'807], arithmetic overflow occurs.
    Integer examples: 
    0
-1
1'024
-900
+4899
4'899
0xffda'34aa
0xffff'ffff'ffff'ffff # equals -1
  • float
    float - double-precision floating-point numbers.
    When writing a number, the ' character can be used as a digit separator.
    A positive number may be preceded by the + character.
    Exponential notation can be used.
    Number examples: 
    0.0
0.1
-0.0
+0.2
153.444
0.1e-10
0.1e+10
100'000'000.0
  • bool
    bool - the values true or false.

  • null
    null - the absence of a value.
    Used when it is necessary to indicate that there is no value, for example if a function returns nothing as a result, it must return null.
    Methods can be called on a constant of type null (more on methods in the obj type), such a call does not evaluate its arguments and always returns null.
    null can be used as the constant for the new operator; in this case, if the key is a token, null will become an object with no fields, otherwise a table with no entries.

    null can be unpacked; when unpacking, specifying any type, in that case all local constants have the value null.
  • byte
    byte - a byte of size 8 bits (architectures with a different byte size are not supported).
    A byte represents a number from the set [0..255]. If during calculations a number goes outside the set of allowed numbers, arithmetic overflow occurs.
    A byte is written as 0bxx, where xx is the hexadecimal representation of the number.
    Byte examples: 
    0b00
0bff
0bfe
  • byte_array
    byte_array - an indexed array whose elements are bytes.
    Array in source code:
    [byte_array byte_1, byte_2, byte_n]
    Indices start from 0.
    An array can have no elements, such an array is called empty.
    Expressions whose result returns a constant of type byte are used as elements.
    The comma between elements can be replaced by a newline character or a ; character.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    Array examples: 
    [byte_array]
[byte_array 0b00]
[byte_array 0b00, foo(a), 0b01 + 0bf1]
[byte_array
     0b00
     foo(a)
     0b01 + 0bf1
]
    Individual elements of the array can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [byte_array name_bound_to_element_with_index_0, name_bound_to_element_with_index_1, name_bound_to_element_with_index_2] = byte_array
    The number of names must not exceed the number of elements in the array.
    Instead of a name, the _ character can be specified; for an element with such an index, no binding to a name will be created.
    To change a byte, the new operator is used; the byte index is passed as the key, and the new byte is passed as the value.

  • time
    time - a type used for working with time.

  • token
    token - identifiers representing a set of characters from 1 to 22, which can be a lowercase Latin letter, a digit (0-9), or an underscore (_).
    After the characters mentioned above, the token may end with one of the following suffixes: !, !!!, ?.
    Token examples: 
    :_
:a
:z
:hello_0123
:some!!!
:0123456789_abcdefghijk!!!
  • fn (function)
    fn - the type for lambda functions; partial application returns a constant of this type.

  • obj (object)
    obj - object in object-oriented programming terminology.
    Objects in source code:
    [obj :field_name_1 = field_1, :field_name_2 = field_2, :field_name_n = field_n]
    An object represents an associative array in which the keys have type token and are field names, and the values of the associative array are the field values.
    An object can have no fields, such an object is called empty.
    The comma between fields can be replaced by a newline character or a ; character.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    Fields cannot be of types from the list: null, box, error, breaker.
    If the field name is known at compile time, null can be used as the value; the field will be absent in that case, but subsequent addition of a field with that name to the object will be more efficient.
    The field with the name _ has a special meaning; such a field (if present) can only be of type obj, and its fields are methods for the object.
    Methods are functions that take as their first argument the object whose method is called, and method arguments start from the second argument of the function.
    If the object from which the method is called has a type other than obj or custom, the arguments of such a method are not evaluated when the method is called.
    If the object from which the method is called has type obj or custom but does not have the specified method, such a situation is an error.
    Unlike functions, methods do not support partial application.
    Method call in source code: object->method_name(argument_1, argument_2, argument_n)
    To add or change a field, the new operator is used; the field name in the form of a token is passed as the key, and the new value is passed as the value.
    If a field with the specified name already existed, the new value replaces the old one; if such a field did not exist, a new field with the specified name is created, and the old value is considered to be null.
    To delete a field, null must be specified as the new field value.
    Object examples: 
    [obj]
[obj :1 = 213]
[obj
     :x = 0.0
     :y = 0.0
]
[obj
     :r = 0b00
     :g = 0b00
     :b = 0b00
]
[obj
     :a = [obj
          :x = 11.3
          :y = 7.4
     ]
     :b = [obj
          :x = -1.8
          :y = 9.11
     ]

     :_ = [obj
          :len = fn (line) ((line:a:x - line:b:x).sqr() + (line:a:y - line:b:y).sqr()).sqrt()
     ]
]
    Individual fields of an object can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [obj name_1 = :field_name_1, name_2 = :field_name_2, name_n = :field_name_n] = obj
    If the field with the name specified during unpacking is absent, the specified name will be bound to the constant null.

  • array
    array - an indexed array.
    Array in source code:
    [array element_1, element_2, element_n]
    Indices start from 0.
    An array can have no elements, such an array is called empty.
    The comma between elements can be replaced by a newline character or a ; character.
    Elements cannot be of types from the list: null, box, error, breaker.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    Array examples: 
    [array]
[array 1]
[array 1, 10.2, "Hello"]
[array
     :hello
     [array 1, 2, 3]
     [obj]
]
    Individual elements of the array can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [array name_bound_to_element_with_index_0, name_bound_to_element_with_index_1, name_bound_to_element_with_index_2] = array
    The number of names must not exceed the number of elements in the array.
    Instead of a name, the _ character can be specified; for an element with such an index, no binding to a name will be created.
    To change elements, the new operator is used; the element index is passed as the key, and the new element is passed as the value.

  • table
    table - an associative array.
    Table in source code:
    [table key_1 = value_1, key_2 = value_2, key_n = value_n]
    For all keys, it must be possible to compute a hash sum using the hash function from the core module, and equality comparison is performed using the binary function ===.
    Values cannot be of types from the list: null, box, error, breaker.
    A table can have no entries, such a table is called empty.
    The comma between multiple keys and values can be replaced by a newline character or a ; character.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    To add or change a value, the new operator is used; the key is passed as the key, and the new value is passed as the value.
    If a value with the specified key already existed, the new value replaces the old one; if a value with that key did not exist, a new value with the specified key is created, and the old value is considered to be null.
    To delete a value, null must be specified as the new value.
    Table examples: 
    [table]
[table "hello" = 1]
[table "hello" = 1, 1 = 1.0]
[table
     "A"   = 'a'
     "b"   = 'B'
     :some = 13
]
    Individual values of the table can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [table name_1 = key_1, name_2 = key_2, name_n = key_n] = table
    If the value with the specified key during unpacking is absent, the specified name will be bound to the constant null.

  • set
    Set in source code:
    [set element_1, element_2, element_n]
    For all elements, it must be possible to compute a hash sum using the hash function from the core module, and equality comparison is performed using the binary function ===.
    A set can have no elements, such a set is called empty.
    The comma between multiple elements can be replaced by a newline character or a ; character.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    Set examples: 
    [set]
[set 1]
[set 1, "hello", :hello]
[set
     "A"
     "b"
     17
     :some
]
    Individual elements of the set can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [set name_1 = element_1, name_2 = element_2, name_n = element_n] = set
    If the element with the specified value during unpacking is absent, the specified name will be bound to the constant null.

  • custom
    custom - a type created using another programming language.
    Such types, similarly to objects, may support methods.

  • box
    box - several constants (elements) grouped into one.
    A box in source code:
    [box element_1, element_2, element_n]
    A box is structured like an indexed array.
    Indices start from 0.
    A box can have no elements, such a box is called empty.
    The maximum number of elements is 16.
    A box cannot be duplicated (stored in several places simultaneously) or reused.
    At any time, a limited number (currently 4) of boxes can exist in the program; therefore, when receiving a box, it is almost always advisable to unpack it.
    The last two points do not apply to boxes that are global constants and empty boxes.
    The main purpose of a box is to pass multiple arguments as one, or to return several constants from a function.
    When creating a box, memory is never allocated on the heap.
    The comma between elements can be replaced by a newline character or a ; character.
    Elements cannot be of type error.
    Before the closing bracket (]), any number of newline or ; characters can be specified.
    Box examples: 
    [box]
[box null]
[box 1, 2, 3, null, [box], "hello"]
[box
     14,1
     null
     [box 1, 2]
     "hello"
]
    Individual elements of a box can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [box name_bound_to_element_with_index_0, name_bound_to_element_with_index_1, name_bound_to_element_with_index_2] = box
    The number of names must not exceed the number of elements in the box.
    Instead of a name, the _ character can be specified; for an element with such an index, no binding to a name will be created.

  • channel
    channel - a type created for passing constants from one thread to another.
    A channel cannot be a global constant.
    A channel cannot be written to a channel.

  • thread
    thread - constants of this type are associated with a running thread, and through them it is possible to interact with that thread.
    Constants of this type cannot be duplicated (stored in several places simultaneously) or reused; also, they cannot be global constants, and they cannot be passed from thread to thread.

  • breaker
    breaker - a type designed for breaking certain functions with which loops are organized (loop, foldl, foldr ...).
    Details on these functions in the documentation for the core module.
    Breaker in source code:
    [breaker constant]
    Constants of this type cannot be duplicated (stored in several places simultaneously) or reused; also, they cannot be global constants, and they cannot be passed from thread to thread.
    At any time, a limited number (currently 8) of breakers can exist in the program.
    When creating a breaker, memory is never allocated on the heap.
    The constant inside the breaker cannot be of type error.
    Before the closing bracket (]), any number of newline or ; characters can be specified.

  • error
    error - constants of this type store information about an error that has occurred.
    Creating an error in source code:
    [error identifier, message, data]
    identifier - a token carrying information about what kind of error occurred.
    message - a string carrying a human-readable error message.
    data - data with technical information about the error; the data must not be of types from the list: box, error, breaker.
    Errors also store information about where they occurred, and in DEBUG mode, additionally information about the sequence of function calls that led to this error.

    If an error occurs in a program, it signals the need to modify the source code; there are only three exceptions to this rule:
    1. if the error that has occurred will lead to the termination of the application
    2. if for some reason you need to call a function that returns an error, but checking that the function will work correctly is extremely difficult or impossible
    3. an incredibly small probability of the error occurring

    When creating a function, one must be guided by the idea that an error occurs only if the developer used it incorrectly.
    For example, if a certain function that accepts flags that affect how the function works receives a flag value that makes no sense, such a situation is an error; whereas if a function that converts a textual representation of a number into a number receives a string that does not contain a textual representation of a number, this is not an error, since such a function can be used on a user-supplied string, and therefore it should return information that the function could not convert the string into a number, without creating an error.
    During development, to determine in which situations a function returns an error and in which it does not, one must be guided by how exactly the function will be used.
    A CORRECTLY WORKING FUNCTION MUST NEVER RETURN AN ERROR!!!

    The program is optimized as if errors never occur in the program, but if one does occur, the program will work correctly.
    If an error is written into any data structure, that structure becomes that error.
    If in the new operator an error is passed as the constant, key, or value, the operator returns that error.
    If in the if operator an error is passed as the predicate, the operator returns that error.
    If in the switch operator an error is passed as the expression on the basis of which the choice is made, the operator returns that error.
    If a method is called on an error, the arguments are not evaluated, and the result is that error.
    If unpacking is performed on a type that is not an error, but the constant being unpacked is an error, then all created local constants become that error.
    If multiple errors are passed to places where the expression would return an error, the error whose expression should have been evaluated first is returned.
    If the main function from the main module returns an error, the program displays that error on the screen.

    The identifier, message, and data can be bound to names, thus creating local constants.
    This process is called unpacking.
    Unpacking can only be done in grouped expressions.
    Local constants created through unpacking follow the same rules as local constants created by the method described in grouped expressions.
    Unpacking in source code:
    let [error identifier, message, data] = error
    Instead of a name, the _ character can be specified, meaning that this component is not needed.
