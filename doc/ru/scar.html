Язык программирования Scar

Строение программ

Программы на языке Scar состоят из модулей.
Каждый модуль имеет имя и состоит из глобальных констант и функций.
Имя модуля должно начинаться со строчной латинской буквы, а остальные символы (если они есть) должны быть либо строчными латинскими буквами, либо цифрами (0-9), либо знаком подчёркивания (_).
Если имя модуля - main, то такой модуль должен содержать функцию с именем main, которая и выполняется в создаваемой программе.
Для создания модуля с помощью компилятора scarlet, необходимо выполнить:

scarlet -n имя_модуля -s имя_файла_с_исходным_кодом_1 имя_файла_с_исходным_кодом_2 имя_файла_с_исходным_кодом_n -o имя_файла_сохраняемого_модуля

Если создаваемый модуль использует другие модули, то файлы с уже созданными модулями передаются с помощью флага -m, пример:

scarlet -n имя_модуля -m имя_файла_модуля_1 имя_файла_модуля_2 имя_файла_модуля_n -s имя_файла_с_исходным_кодом -o имя_файла_сохраняемого_модуля

Файлы с исходным кодом имею расширение - scar, а файлы с модулем - mscar.
Если модуль имеет имя main, то вместо файла с модулем, сохраняется файл с промежуточным представлением. На данный момент это исходной код на языке C предназначенный только для компилятора clang.

Строение файлов с исходным кодом

Файл с исходным кодом может содержать следующие элементы:
  • Однострочный комментарий. Начинается с символа # и заканчивается концом строки.
  • Многострочный комментарий. Начинается с символов #* и заканчивается символами *#.
  • Комментарий документации. Начинается с символов #@ и заканчивается символами @#. Располагается перед глобальной константой или функцией (но перед их атрибутами).
  • Атрибуты функции или глобальной константы. Располагаются перед глобальной константой или функцией.
  • Глобальная константа. (Подробности в разделе Глобальные константы)
  • Функция. (Подробности в разделе Функции)
Для форматирования кода не допускается символ табуляции.
Исходный код должен быть в кодировке ctf8. (ASCII совместимая)

Глобальные константы

Глобальные константа - это константа имеющая имя, значение которой доступно по данному имени в выражениях других констант и функций. (Подробности о выражениях в разделе Выражения)
Объявление константы в исходном коде: 
`атрибуты`
let имя_константы = значение
Указание атрибутов не является обязательным.
Атрибуты:
  • export - константа доступна из любого модуля, без этого атрибута - только в модуле в котором объявлена.
Имя константы должно начинаться со строчной латинской буквы или знака подчёркивания (_), а остальные символы (если они есть) должны быть либо строчными латинскими буквами, либо цифрами (0-9), либо знаком подчёркивания.
Имя константы не может состоять только из одного знака подчёркивания.
В одном модуле две константы не могут иметь одно имя.

В качестве значения указывается выражение, результат которого и является значением константы.
Если выражение не может быть вычислено во время компиляции, то оно вычисляется до выполнения функции main из модуля main.
Выражение не может использовать потоки.
Значение глобальной константы вычисляется только если функция main из модуля main, напрямую или опосредованно использует данную константу.
Если значение глобальной константы имеет тип error, то программа немедленно останавливается с сообщением о данной ошибке.

Функции

Функция - это выражение в, котором могут быть обращения к константам по имении, а значения этих констант буду указанны при вызове функции. (Пример: fn y(x) x + 1) (Подробности о выражениях в разделе Выражения)
Константы, значение которых указываются при вызове функции, называются аргументами функции, а их имена внутри функции называются параметрами функции.
Выражение называется телом функции.
Тело функции вычисляется при вызове функции, заменяя параметры функции вычисленными значениями аргументов.
Аргументы вычисляются в том же порядке, в каком они передаются в функцию.
Объявление функции в исходном коде: 
`атрибуты`
fn имя_функции(параметр_1, параметр_2, параметр_n) выражение
Указание атрибутов не является обязательным.
Если необходимо перечислить несколько атрибутов, то они разделяются одним или несколькими пробелами.
Атрибуты:
  • cold - функция оптимизируется так, как будто она никогда не выполнится, но если это произойдёт, она выполнится корректно.
  • export - функция доступна из любого модуля, без этого атрибута - только в модуле в котором объявлена.
  • external - реализация данной функции сделана на другом языке программирования. В данном случае тело функции не указывается.
  • hot - программа оптимизируется так, как будто данная функция будет часто выполняться. Указание данного атрибута в функции которая вызывается редко, может приводить к ухудшению производительности программы.
  • noinline - иногда вызов функции может быть заменён на тело функции с подставленными аргументами, данный атрибут отключает такое поведение для данной функции.
Атрибуты hot и cold не могут указываться одновременно.

Имя функции должно начинаться со строчной латинской буквы или знака подчёркивания (_), а остальные символы (если они есть) должны быть либо строчными латинскими буквами, либо цифрами (0-9), либо знаком подчёркивания.
Так же в имени функции допускается один из следующих суффиксов - !, !!!, ?.
Значения суффиксов в имени функции:
  • ? - имя функции является вопросом, а результат вычисления функции даст на него ответ.
  • ! - имя функции является утверждением, и если утверждение не верно, то функция изменит состояние приложения таким образом, что оно станет верным, либо вернёт ошибку.
  • !!! - функция является небезопасной и пользоваться ей следует с осторожностью.
Имя функции не может быть именем из указанного списка: case, default, else, fn, guard, if, let, new, not, scarlet__erase_info, scarlet__erase_value, switch, test, unreachable, _
В одном модуле две функции не могут иметь одно имя.

Имя параметра функции должно начинаться со строчной латинской буквы или знака подчёркивания (_), а остальные символы (если они есть) должны быть либо строчными латинскими буквами, либо цифрами (0-9), либо знаком подчёркивания.
Имя параметра функции не может быть именем из указанного списка: case, default, else, false, fn, if, let, new, not, null, switch, true
Если в место имени параметра указывается _, то это означает что функции не нужно значение данного аргумента.
В одной функции два параметра не могут иметь одно имя.
Количество параметров функции не должно быть больше 16.

Выражения

Выражение - вычисление либо действие возвращающие результат.
Если в каком либо месте исходного кода ожидается выражение, то перед ним может быть сколь угодно много символов новой строки или символов ; (за исключением случаев, когда данные символы заменяют запятую). (Пример: 2 + ;; 2 , то же что и 2 + 2 )
У выражений есть приоритет (число от 0 до 10).
Если выражение состоит их нескольких выражений, то данное выражение является тем выражением, приоритет которого выше.
Если есть два выражения с одинаковым приоритетом, то считается что выражение ближе к началу исходного выражения, имеет меньший приоритет.
Для упрощения, далее в тексте процесс выяснения каким выражением является данное выражение, будет называться "выяснением корневого выражения". Пример:
1 + 2 + 4
Данное выражение является бинарной функцией + (о бинарных функциях далее), которая принимает два аргумента: первый 1 + 2, второй 4.

Список возможных выражений:
  • Бинарная функция
    Бинарные функции - функции принимающие два аргумента, а имя функции находится между ними.
    В функцию не обязательно передавать сразу все аргументы сразу, можно передать часть аргументов, а в место остальных передать _, в этом случае вызов функции вернёт функцию, которая принимает столько аргументов, сколько было передано _, а затем выполнит исходную функцию заменив _ на переданные аргументы в том же порядке. Такое применение функции называется частичным.
    При частичном применении, передаваемые аргументы не могут быть следующих типов: box, breaker, thread, error. (Подробности о типах в разделе Типы)

    Примеры:
    _ + 1 , то же что и fn (a) a + 1
    _ + _ , то же что и fn (a, b) a + b

    Список бинарных функций и их приоритет:
    Функции Приоритет
    [] : [,] 0
    * / % 3
    + ++ - 4
    << >> >>> 5
    < > <= >= 6
    == /= === =/= 7
    & 8
    ^ 9
    | 10
    Какие действия выполняю данные функции, описано в документации к модулю core.

  • Бинарный оператор
    Бинарные операторы - то же что и бинарные функции, но без возможности частичного применения и с ленивым вычислением аргументов.
    Список бинарных операторов и их приоритет:
    Оператор Приоритет Выполняемое действие
    <? 1 Вычисляет левое выражение, если оно null, то вычисляет правое выражение и его результат является результатом всего выражения, иначе не вычисляет правое выражение и результатом всего выражения является результат левого выражения. (Подробности о null в разделе Типы)
    && 8 Вычисляет левое выражение, если оно равно true, то вычисляет правое выражение и его результат является результатом всего выражения, иначе не вычисляет правое выражение и результатом всего выражения является false. Оба выражения должны иметь тип bool. (Подробности о false, true и типе bool в разделе Типы)
    || 10 Вычисляет левое выражение, если оно равно false, то вычисляет правое выражение и его результат является результатом всего выражения, иначе не вычисляет правое выражение и результатом всего выражения является true. Оба выражения должны иметь тип bool. (Подробности о false, true и типе bool в разделе Типы)

  • Вызов функции
    Функцию в исходном коде можно вызвать двумя способами:
    1. имя_функции(аргумент_1, аргумент_2, аргумент_n)
    2. аргумент_1.имя_функции(аргумент_2, аргумент_n)
    Имя функции можно указать двумя способами:
    1. короткий - просто указывается имя функции. Допускается если только в одном модуле есть функция с указанным именем.
    2. полный - имя функции указывается в формате @имя_модуля.имя_функции.
    Как и в бинарных функциях, можно использовать частичное применение.
    Если аргументов несколько, запятая между ними может быть заменена на символ новой строки или символ ;.
    Пример:
    foo(1, 2, 3, 4)
    то же что и 
    foo(
     1
     2; 3, 4
)
    Аргументы переданные после указания имени функции (в скобках), не участвуют в том же выяснении корневого выражения, что и данный вызов функции.
    Приоритет - 0.

  • Оператор check
    Оператор работает как функция, но без возможности частичного применения.
    Оператор принимает один аргумент и если он имеет тип error, то приложение немедленно останавливается с информацией о том, в каком месте исходного кода использовался оператор и что за ошибка была в аргументе, иначе результатом оператора является его аргумент. (Подробности о типе error в разделе Типы)

    Приоритет - 0.

  • Оператор guard
    Оператор работает как функция, но без возможности частичного применения.
    Оператор принимает один аргумент и если он равен true, то результатом оператора является true, иначе приложение немедленно останавливается с информацией о том, в каком месте исходного кода использовался оператор. (Подробности о true в разделе Типы)

    Приоритет - 0.

  • Оператор test
    Оператор работает как функция, но без возможности частичного применения.
    Оператор принимает один аргумент.
    Если было хотя бы одно использование оператора test, то после окончания работы приложения, отображается информация о том как много раз был вызван оператор, сколько раз аргумент был равен true, а также информация о том, в каком месте исходного кода использовался оператор с аргументом отличным от true. Результатом оператора является null. Подробности о true и null в разделе Типы.

    Приоритет - 0.

  • Оператор unreachable
    Оператор не принимает аргументов.
    При вызове оператора приложение немедленно останавливается с информацией о том, в каком месте исходного кода использовался оператор.

    Приоритет - 0.

  • Константы true и false (Подробности в разделе Типы)

    Приоритет - 0.

  • Числовая константа
    (Подробности в разделе Типы)

    Приоритет - 0.

  • Символ
    (Подробности в разделе Типы)

    Приоритет - 0.

  • Константа null
    (Подробности в разделе Типы)

    Приоритет - 0.

  • Строка
    (Подробности в разделе Типы)

    Приоритет - 0.

  • Токен
    (Подробности в разделе Типы)

    Приоритет - 0.

  • Глобальная константа
    Получение значения глобальной константы в исходном коде:
    @имя_модуля.имя_константы

    Приоритет - 0.

  • Сгруппированное выражения
    Несколько выражений могут быть сгруппированы в одно выражения, в этом случае выражения выполняются в том порядке в котором указанны, а результатом сгруппированных выражений, является результат последнего выражения.
    Для группировки выражений, необходимо заключить все выражения в фигурные скобки ({}), разделив выражения символом новой строки или символом ;.
    В сгруппированных выражениях можно связать имя с константой, для этого в качестве одного из выражений нужно написать let имя_константы = значение, после этого до конца сгруппированного выражения, по данному имени будет доступен результат выражения.
    Если с данным именем уже была связанна другая константа или данное имя является параметром функции, то до конца сгруппированного выражения, по данному имени будет доступна новая константа.
    Правила для имен констант, такие же как и для параметров функции.
    Нельзя связывать имена с константами в последнем выражении.
    Связывать константы с именами можно так же с помощью распаковки, подробности о распаковке в разделе Типы.

    Сгруппированные выражения участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Локальная константа
    Локальная константа - константа связанная с каким либо именем, но не принадлежащая какому либо модулю.
    Аргументы функций в самих функциях являются локальными константами.
    Константы которые были связанны с именем в сгруппированных выражениях, являются локальными константами.
    Для обращения к значению константы, нужно просто указать её имя.
    Приоритет - 0.

  • Вложенное выражение
    Выражение можно обернуть в круглые скобки (), в этом случае выражение участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Оператор new
    Если какая либо константа хранит в себе другие константы, то с помощью оператора new, можно создать новую константу, такую же как данная константа, но с новыми значениями внутренних констант.
    Оператор new имеет две формы:
    1. new (константа)[ключ_1][ключ_2][ключ_3] модификатор значение
    2. new (константа) {[ключ_1][ключ_2][ключ_3] модификатор_1 значение_1; [ключ_4][ключ_5][ключ_6] модификатор_2 значение_2}
    Константа на основании которой, будет создан результат, записывается в круглых скобках после new.
    Ключи - это значения с помощью которых определяется какая часть константы будет изменена, то как именно это происходит зависит от типа к которому ключ применяется.
    (Подробности в разделе Типы)
    Если в качестве ключа для оператора new указывается токен, непосредственно в виде :токен, то квадратные кавычки ([]) указывать не обязательно.
    (Подробности о токенах в разделе Типы)
    ключ_1 указывает на то, какая часть константы будет изменена, а ключ_2 какая часть константы на которую указывает ключ_1 будет изменена и т.д.

    Типы модификаторов:
    Модификатор Назначение
    <= заменяет старое значение на новое
    <<= то же что и <=, но оператор возвращает не только новое значение константы, но и старое значение изменённой части (значение обвёрнуты в тип box подробности в разделе Типы)
    => то же что и <=, но в качестве нового значения указывается функция, которая принимает в качестве аргумента старое значение изменяемой части, а возвращает его новое значение
    =>> то же что и =>, но оператор возвращает не только новое значение константы, но и старое значение изменённой части (значение обвёрнуты в тип box подробности в разделе Типы)

    Выражение, результат которого является новым значением внутренней константы, берётся до тех пор, пока является верным. Пример:
    new ([array x])[0] <= 12 + 1
    Данное выражение вернёт [array 13]

    Порядок вычисления выражений:
    1. ключи
    2. новые значения
    3. константа

    Во второй форме оператора точка с запятой может заменятся на символ новой строки
    Вторая форма оператора, является удобной формой записи применения нескольких использований подряд оператора в первой форме. Пример: 
    new (a) {
     [1] <= 2
     [f][x][13] <= f
}
    то же что и 
    new (new (a)[1] <= 12)[f][x][13] <= f
    Во второй форме оператора, модификаторы <<= и =>> могут использоваться только в последней модификации.

    Оператор new участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Оператор if
    Оператор if используется для вычисления одного из двух выражений.
    Оператор в исходном коде:
    if предикат выражение_1 else выражение_2

     Для разделения предиката и выражения_1 нужно использовать символ новой строки или символ ;.
    Если выражения_1 является сгруппированным выражением, то разделять предикат и выражение_1 не обязательно.
    Перед else может быть сколь угодно много символов новой строки или символов ;.
    Выражение после else берётся до тех пор, пока является верным.

    Оператор вычисляет предикат, если его результат равен true, то вычисляется выражение_1 и его результат является результатом всего выражения, иначе вычисляется выражение_2 и его результат является результатом всего выражения. Предикат должны иметь тип bool. (Подробности о true и типе bool в разделе Типы)

    Оператор if участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Оператор switch
    Оператор switch используется для вычисления одного из нескольких выражений.
    Оператор в исходном коде:
    switch выражение_0 case значение_1, значение_2: вырежение_1 case значение_m: выражение_n: default: выражение_d

    Оператор вычисляет выражение_0, если его результат равен одному из значений идущих после case но до двоеточия, то вычисляется выражение после двоеточия и его результат, является результатом всего выражения, иначе вычисляется выражение после default: и его результат, является результатом всего выражения.
    После case может следовать несколько выражений подряд, разделённых запятой (которая может быть заменена на символом новой строки или символом ;).
    Выражение после default: берётся до тех пор, пока является верным.
    Для всех выражения после case, должна быть возможность вычислить хэш сумму с помощью функции hash из модуля core, а сравнение на равенство происходит с помощью бинарной функции ===.
    В одном операторе switch, для двух любых выражений после любого case, бинарная функция === должна вернуть false. (Подробности о false в разделе Типы)
    Если выражение после case не может быть вычислено во время компиляции, то оно вычисляется до выполнения функции main из модуля main.

    Оператор switch участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Лямбда функция
    Лямбда функция - функция не имеющая имени.
    Лямбда функция отличается от обычной только отсутствием имени и отсутствием принадлежности к какому либо модулю.
    Лямбда функции в исходном коде:
    fn (параметр_1, параметр_2, параметр_n) выражение

    Вызов лямбда функции в исходном коде:
    (выражение)(аргумент_1, аргумент_2, аргумент_n)

    Для лямбда функций, как и для обычных функций, можно использовать частичное применение.
    В теле лямбда функции можно обращаться к локальным константам которые были объявлены за пределами лямбда функции, такие константы не могут быть следующих типов: box, breaker, thread, error. (Подробности о типах в разделе Типы)
    Если в теле лямбда функции есть обращение к локальным константам которые были объявлены за пределами лямбда функции и есть параметр данной лямбда функции с таким же именем, то в таком случае данное имя будет интерпретировано как параметр функции.
    Выражение берётся до тех пор, пока является верным.

    Лямбда функция участвуют в выяснении корневого выражения как одно целое.
    Приоритет - 0.

  • Вызов метода

    Приоритет - 0.

  • Функция not Функция в исходном коде:
    not выражение

    Функция вычисляет выражение и если его результат равен true, то результат функции - false, а если результат выражения равен false, то результат функции - true.
    Функция принимает только аргументы типа bool. (Подробности о false, true и типе bool в разделе Типы)

    Приоритет - 1.

Типы

Тип - характеристика константы, исходя из которой программа определяет что это за константа и как с ней работать.
Список типов:
  • string (строка)
    string - текст.
    В тексте могут отсутствовать какие либо символы, такая строка называется пустой.
    Строки в языке scar использую свою кодировку символов под названием corsar.
    Строка начинается и заканчивается символом ".
    Строки поддерживают управляющие последовательности, которые начинаются с символа \.
    Управляющие последовательности:
    управляющая последовательность значение
    \n символ новой строки
    \t символ табуляции
    \' символ '
    \" символ "
    \\ символ \
    \xxxxxx символ с кодом xxxxxx, где xxxxxx - шестнадцатеричный код символа в кодировке corsar

    Перед открывающей скобкой можно указать букву r, в этом случае управляющие последовательности будут игнорироваться.
    Перед открывающей скобкой можно указать букву u, в этом случае строка будет иметь тип не string, а byte_array (о нём позже), и хранить байты со строкой аналогичной данной, но в кодировке utf8, а управляющая последовательность \xxxxxx будет означать код в юникоде.
    Если указана буква u, то после неё можно указать букву z, в этом случае в конец строки добавится байт 0 (ноль).
    Перед u или uz можно указать r, в этом случае управляющие последовательности будут игнорироваться.

    Примеры строк: 
    ""
"Hello !!!"
"Char \000100"
"New line \n"
r"not new line \n"
"\""
u""
uz""
u"Hello"
u"\000100"
ru"\000100"
ruz"\000100"
    Для изменения символа, используется оператор new, в качестве ключа передаётся индекс символа (начинается с 0), а в качестве значения - новый символ.

  • char (символ)
    char - один символ текста.
    Символ начинается и заканчивается символом '.
    Символы поддерживают управляющие последовательности, таким же образом как и строки.
    Перед открывающей скобкой можно указать букву u, в этом случае символ будет иметь тип не char, а int (о нём позже), и хранит код символа аналогичного данновму но в юникоде, а управляющая последовательность \xxxxxx будет означать код в юникоде.
    Примеры символов: 
    'a'
'z'
' '
'\000100'
'\n'
'\''
u'a'
u'H'
u'\000100'
  • int
    int - целое число из множества [-9'223'372'036'854'775'808 - 9'223'372'036'854'775'807] (-2^63 - 2^63-1).
    При записи числа можно использовать символ ', который используется как разделитель разрядов.
    Перед положительным числом может быть символ +.
    Если при расчётах число выходит за множество допустимых чисел, то происходит арифметическое переполнение.
    Число может быть записано в шестнадцатеричной системе счисления, для этого перед числом нужно написать 0x (после символов - или +, если они есть).
    Шестнадцатеричная записи может иметь значения от -0xffff'ffff'ffff'ffff до 0xffff'ffff'ffff'ffff (-(2^64-1) - 2^64-1), но при этом если число не принадлежит множеству [-9'223'372'036'854'775'808 - 9'223'372'036'854'775'807], то происходит арифметическое переполнение.
    Примеры целых чисел: 
    0
-1
1'024
-900
+4899
4'899
0xffda'34aa
0xffff'ffff'ffff'ffff # -1
  • float
    float - числа с плавающей точкой двойной точности.
    При записи числа можно использовать символ ', который используется как разделитель разрядов.
    Перед положительным числом может быть символ +.
    Можно использовать экспоненциальную форму записи.
    Примеры чисел: 
    0.0
0.1
-0.0
+0.2
153.444
0.1e-10
0.1e+10
100'000'000.0
  • bool
    bool - значения true (истина) или false (ложь).

  • null
    null - отсутствие значения.
    Используется если нужно указать, что никакого значения нет, например если функция ничего не возвращает в качестве результата, то она должна вернуть null.
    Из константы типа null можно вызывать методы (подробнее о методах в типе obj), такой вызов не вычисляет свои аргументы и всегда возвращает null.
    null может использоваться в качестве константы для оператора new, в этом случае если ключ является токеном, то null станет объектом без полей, иначе таблицей без элементов.

    null можно распаковывать, при распаковке указав любой тип, в этом случае все локальные константы имеют значение null.
  • byte
    byte - байт размером 8 бит (архитектуры с другим размером байта не поддерживаются).
    Байт представляет из себя число из множества [0..255]. Если при расчётах число выходит за множество допустимых чисел, то происходит арифметическое переполнение.
    Байт записывается как 0bxx, где xx - шестнадцатеричное представление числа.
    Примеры байт: 
    0b00
0bff
0bfe
  • byte_array
    byte_array - индексный массив элементы которого являются байтами.
    Массив в исходном коде:
    [byte_array байт_1, байт_2, байт_n]
    Индексы начинаются с 0.
    Массив может не иметь элементов, такой массив называется пустым.
    В качестве элементов используются выражения, результат которых возвращает константу типа byte.
    Запятая между элементами может быть заменена на символ новой строки или символ ;.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Примеры массивов: 
    [byte_array]
[byte_array 0b00]
[byte_array 0b00, foo(a), 0b01 + 0bf1]
[byte_array
     0b00
     foo(a)
     0b01 + 0bf1
]
    Отдельные элементы массива можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [byte_array имя_связанное_с_элементом_с_индексом_0, имя_связанное_с_элементом_с_индексом_1, имя_связанное_с_элементом_с_индексом_2] = byte_array
    Количество имён не должно превышать количество элемент в массиве.
    В место имени, можно указать символ _, для элемента с таким индексом связь с именем не будет создана.
    Для изменения байта, используется оператор new, в качестве ключа передаётся индекс байта, а в качестве значения - новый байт.

  • time
    time - тип используемый для работы с временем.

  • token (токен)
    token - идентификаторы представляющие из себя набор символов от 1 до 22, которые могут быть строчной латинской буквой, цифрой (0-9), знаком подчёркивания (_).
    После указанных выше символов, токен может заканчиваться одним из следующих суффиксов: !, !!!, ?.
    Примеры токенов: 
    :_
:a
:z
:hello_0123
:some!!!
:0123456789_abcdefghijk!!!
  • fn (функция)
    fn - тип для лямбда функций, частичное применение возвращает константу данного типа.

  • obj (объект)
    obj - объект в терминологии объектно ориентированного программирования.
    Объекты в исходном коде:
    [obj :имя_поля_1 = поле_1, :имя_поля_2 = поле_2, :имя_поля_n = поле_n]
    Объект представляет из себя ассоциативный массив в котором ключи имеют тип token и являются именами полей, а значения ассоциативного массива являются значениями полей.
    Объект может не иметь полей, такой объект называется пустым.
    Запятая между полями может быть заменена на символ новой строки или символ ;.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Поля не могут иметь тип из списка: null, box, error, breaker.
    Если имя ключа известно при компиляции, то в качестве значения можно использовать null, поле при этом будет отсутствовать, но последующее добавление в объект поля с данным именем будет более эффективным.
    Поле с именем _ имеет особое значение, такое поле (если оно есть) может быть только типа obj и его поля являются методами для объекта.
    Методы являются функциями, которые в качестве первого аргумента принимают объект, метод которого вызывается, а аргументы метода начинаются со второго аргумента функции.
    Если объект из которого вызывается метод имеет тип отличный от типов obj или custom, то аргументы такого метода не вычисляются при вызове метода.
    Если объект из которого вызывается метод имеет тип obj или custom, но не имеет указанного метода, то такая ситуация является ошибкой.
    В отличие от функции, методы не поддерживают частичное применение.
    Вызов метода в исходном коде: объект->имя_метода(аргумент_1, аргумент_2, аргумент_n)
    Для добавление или изменения поля, используется оператор new, в качестве ключа передаётся имя поля в виде токена, а в качестве значения - новое значение.
    Если поле с указанным именем уже существовало, то новое значение заменит старое, а если такого поля не было, просто создастся новое поле с указанным именем, а старым значением будет считаться null.
    Чтобы удалить поле, необходимо в качестве нового значения поля указать null.
    Примеры объектов: 
    [obj]
[obj :1 = 213]
[obj
     :x = 0.0
     :y = 0.0
]
[obj
     :r = 0b00
     :g = 0b00
     :b = 0b00
]
[obj
     :a = [obj
          :x = 11.3
          :y = 7.4
     ]
     :b = [obj
          :x = -1.8
          :y = 9.11
     ]

     :_ = [obj
          :len = fn (line) ((line:a:x - line:b:x).sqr() + (line:a:y - line:b:y).sqr()).sqrt()
     ]
]
    Отдельные поля объекта можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [obj имя_1 = :имя_поля_1, имя_2 = :имя_поля_2, имя_n = :имя_поля_n] = obj
    Если поле с именем указанном при распаковке - отсутствует, то указанное имя будет связанно с константой null.

  • array
    array - индексный массив.
    Массив в исходном коде:
    [array элемент_1, элемент_2, элемент_n]
    Индексы начинаются с 0.
    Массив может не иметь элементов, такой массив называется пустым.
    Запятая между элементами может быть заменена на символ новой строки или символ ;.
    Элементы не могут иметь тип из списка: null, box, error, breaker.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Примеры массивов: 
    [array]
[array 1]
[array 1, 10.2, "Hello"]
[array
     :hello
     [array 1, 2, 3]
     [obj]
]
    Отдельные элементы массива можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [array имя_связанное_с_элементом_с_индексом_0, имя_связанное_с_элементом_с_индексом_1, имя_связанное_с_элементом_с_индексом_2] = array
    Количество имён не должно превышать количество элемент в массиве.
    В место имени, можно указать символ _, для элемента с таким индексом связь с именем не будет создана.
    Для изменения элементов, используется оператор new, в качестве ключа передаётся индекс элемента, а в качестве значения - новый элемент.

  • table (таблица)
    table - ассоциативный массив.
    Таблица в исходном коде:
    [table ключ_1 = значение_1, ключ_2 = значение_2, ключ_n = значение_n]
    Для всех ключей должна быть возможность вычислить хэш сумму с помощью функции hash из модуля core, а сравнение на равенство происходит с помощью бинарной функции ===.
    Значения не могут иметь тип из списка: null, box, error, breaker.
    Таблица может не иметь значений, такая таблица называется пустой.
    Запятая между несколькими ключами и значениями, может быть заменена на символ новой строки или символ ;.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Для добавление или изменения значения, используется оператор new, в качестве ключа передаётся ключ, а в качестве значения - новое значение.
    Если значение с указанным ключом уже существовало, то новое значение заменит старое, а если значения с таким ключом не существовало, просто создастся новое значение с указанным ключом, а старым значением будет считаться null.
    Чтобы удалить значение, необходимо в качестве нового значения указать null.
    Примеры таблиц: 
    [table]
[table "hello" = 1]
[table "hello" = 1, 1 = 1.0]
[table
     "A"   = 'a'
     "b"   = 'B'
     :some = 13
]
    Отдельные значения таблицы можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [table имя_1 = ключ_1, имя_2 = ключ_2, имя_n = ключ_n] = table
    Если значение с указанным ключом при распаковке - отсутствует, то указанное имя будет связанно с константой null.

  • set (множество)
    Множество в исходном коде:
    [set элемент_1, элемент_2, элемент_n]
    Для всех элементов должна быть возможность вычислить хэш сумму с помощью функции hash из модуля core, а сравнение на равенство происходит с помощью бинарной функции ===.
    Множество может не иметь элементов, такое множество называется пустым.
    Запятая между несколькими элементов, может быть заменена на символ новой строки или символ ;.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Примеры множеств: 
    [set]
[set 1]
[set 1, "hello", :hello]
[set
     "A"
     "b"
     17
     :some
]
    Отдельные элементы множества можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [set имя_1 = элемент_1, имя_2 = элемент_2, имя_n = элемент_n] = set
    Если значение с указанным ключом при распаковке - отсутствует, то указанное имя будет связанно с константой null.

  • custom
    custom - тип созданный с помощью другого языка программирования.
    Такие типы, по аналогии с объектами, могут поддерживать методы.

  • box (бокс)
    box - несколько констант (элементов) сгруппированных в одну.
    Бокс в исходном коде:
    [box элемент_1, элемент_2, элемент_n]
    Бокс устроен как индексный массив.
    Индексы начинаются с 0.
    Бокс может не иметь элементов, такой бокс называется пустым.
    Максимальное количество элементов - 16.
    Бокс нельзя дублировать (хранить одновременно в несколько местах) или переиспользовать.
    Одновременно в программе может существовать ограниченное число (на данный момент 4) боксов, поэтому при получении бокса, практически всегда его стоит распаковать.
    Последние два пункта не касаются боксов являющихся глобальными константами и пустых боксов.
    Основное назначение бокса, передать несколько аргументов как один, или вернуть из функции несколько констант.
    При создании бокса, никогда не выделяется память на куче.
    Запятая между элементами может быть заменена на символ новой строки или символ ;.
    Элементы не могут иметь тип error.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.
    Примеры боксов: 
    [box]
[box null]
[box 1, 2, 3, null, [box], "hello"]
[box
     14,1
     null
     [box 1, 2]
     "hello"
]
    Отдельные элементы бокса можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [box имя_связанное_с_элементом_с_индексом_0, имя_связанное_с_элементом_с_индексом_1, имя_связанное_с_элементом_с_индексом_2] = box
    Количество имён не должно превышать количество элементов в боксе.
    В место имени, можно указать символ _, для элемента с таким индексом связь с именем не будет создана.

  • channel (канал)
    channel - тип созданный для передачи констант из одного потока, в другой.
    Канал не может быть глобальной константой.
    Канал нельзя записывать в канал.

  • thread
    thread - константы данного типа связаны с запущенным потоком и с их помощью с этим потоком можно взаимодействовать.
    Константы данного типа нельзя дублировать (хранить одновременно в несколько местах) или переиспользовать, так же они не могут быть глобальными константами, и их нельзя передавать из потока в поток.

  • breaker (прерыватель)
    breaker - тип предназначенный для прерывания некоторых функций, с помощью которых организуются циклы (loop, foldl, foldr ...).
    Подробности к данным функциями в документации к модулю core.
    Прерыватель в исходном коде:
    [breaker константа]
    Константы данного типа нельзя дублировать (хранить одновременно в несколько местах) или переиспользовать, так же они не могут быть глобальными константами, и их нельзя передавать из потока в поток.
    Одновременно в программе может существовать ограниченное число (на данный момент 8) прерывателей.
    При создании прерывателя, никогда не выделяется память на куче.
    Константа внутри прерывателя не может иметь тип error.
    Перед закрывающей кавычкой (]), можно указывать сколько угодно символов новой строки или символов ;.

  • error (ошибка)
    error - константы данного типа, хранят информацию о произошедшей ошибке.
    Создание ошибки в исходном коде:
    [error идентификатор, сообщение, данные]
    идентификатор - токен носящий информацию о том, какого рода ошибка произошла.
    сообщение - строка носящая человеко-читаемое сообщение об ошибки.
    данные - данные с технической информацией об ошибке, данные не должны иметь тип из списка: box, error, breaker.
    Ошибки также хранят информацию о том где они произошли, а в DEBUG режиме, еще и информацию о последовательности вызовов функций приведший к этой ошибке.

    Если ошибка произошла в программе, это сигнализирует о необходимости изменения исходного кода, из этого правила есть только три исключения:
    1. если произошедшая ошибка приведёт к завершению работы приложения
    2. если по каким либо причинам вам нужно вызвать функцию которая возвращает ошибку, но при этом проверка на то, что функция отработает корректно - крайне затруднительна или невозможна
    3. невероятно маленькая вероятность возникновения ошибки

    При создании функции, нужно ориентироваться на то, что ошибка происходит только если разработчик использовал её не корректно.
    Например если в некую функцию, которая принимает флаги от которых зависит как функция работает, было передано значение флагов которое не имеет смысла, то такая ситуация является ошибкой, а если в функцию, которая превращает текстовое представление числа в число, передана строка которая не содержит текстовое представление числа, то это не ошибка, так как такая функция может использоваться на переданной пользователем строке, по этому нужно возвращать информацию о том, что функция не смогла превратить строку в число, не создавая ошибку.
    При разработке для определения в каких ситуациях функция возвращает ошибку, а в каких нет, нужно ориентироваться на то, как именно функция будет использоваться.
    КОРРЕКТНО ОТРАБОТАВШАЯ ФУНКЦИЯ, НИ В КОЕМ СЛУЧАЕ НЕ ДОЛЖНА ВОЗВРАЩАТЬ ОШИБКУ!!!

    Программа оптимизируется таким образом, как если в программе никогда не происходит ошибок, но если это случится, то программа отработает корректно.
    Если записать ошибку в любую структуру данных, то такая структура станет данной ошибкой.
    Если в оператор new в качестве константы, ключа или значения передать ошибку, то оператор вернёт данную ошибку.
    Если в оператор if в качестве предикаты передать ошибку, то оператор вернёт данную ошибку.
    Если в оператор switch в качестве выражения на основании которого делается выбор передать ошибку, то оператор вернёт данную ошибку.
    Если метод вызывается из ошибки, то аргументы не вычисляются, а результатом будет данная ошибка.
    Если происходит распаковка типа не являющегося ошибкой, но распаковываемая константа является ошибкой, то все созданные локальные константы становятся данной ошибкой.
    Если несколько ошибок переданы в такие места, в которых выражение вернёт ошибку, то возвращается ошибка, выражение которой должно вычистился первым.
    Если функция main из модуля main возвращает ошибку, то программа выводит на экран данную ошибку.

    идентификатор, сообщение и данные можно связать с именами, создав таким образом локальные константы.
    Такой процесс называется распаковкой.
    Распаковку можно делать только в сгруппированных выражениях.
    Созданные с помощью распаковки локальные константы, подчиняются тем же правилам, что и локальные константы созданные способом описанным в сгруппированных выражениях.
    Распаковка в исходном коде:
    let [error идентификатор, сообщение, данные] = error
    В место имени, можно указать символ _, что означает - данный компонент не нужен.
